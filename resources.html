◆ PORTFOLIO TARGETS — BACKEND & AI ENGINEERING
Project 1: Production Backend API
EASY Python · FastAPI · PostgreSQL · Docker
Clean architectural layout: dynamic dependency injection, domain models, database repositories, custom Exception handlers. Learns robust production-grade API architecture.
Redis Strategy
None in MVP context; pure relational persistence structure.
Async Setup
Using async session models (async/await) completely to prevent connection blockages.
Scaling Concerns
DB locking bottlenecks: implement connection pooling, configure proper query limits, index tables carefully.
Observability
Integrated standard logging to trace endpoint load metrics and write error telemetry directly.
API Endpoints
  • • POST /users — create account with validation
  • • POST /login — stateless token generation
  • • GET /items — dynamic query + pagination
  • • PUT /items/{id} — atomic record modification
Dev Checks
  • • Write structural Pydantic schemas with strict regex validation
  • • Configure unified error middleware for unhandled exceptions
  • • Deploy Docker containers with isolated DB databases
Prod Stack Concern CORS configurations, input boundaries sanitization, database connection timeouts.
Project 2: Scalable URL Shortener
EASY Python · FastAPI · Redis · Docker
High-speed redirection engine. Focuses on in-memory operations, rapid lookups, custom analytics writing, and route protection.
Redis Strategy
String storage for direct redirects, Hashes for analytics, rate-limiting counters.
Async Setup
Non-blocking Redis await operations for single-digit millisecond latency responses.
Scaling Concerns
Massive simultaneous reads: Redis as primary read-path, offload analytics to lazy writers.
Observability
Sliding window click tracking. Monitor cache hit-ratio to verify optimization quality.
API Endpoints
  • • POST /shorten — generate short slug with custom TTL
  • • GET /{slug} — performance-centric redirect + analytics
  • • GET /analytics/{slug} — hit maps and visitor metadata
Dev Checks
  • • Implement sliding rate-limiter in Redis to prevent API abuse
  • • Build lightweight dashboard displaying URL performance maps
  • • Dockerize API and Redis setup under local container networking
Prod Stack Concern High-concurrency cache eviction policies, thundering herds mitigation, securing redirects from infinite loops.
Project 3: Async File Processor
MEDIUM Python · FastAPI · Redis · Celery · Docker
Distributed queue architecture. FastAPI acts as gateway, passing heavy jobs to Redis broker, which hands them to a decoupled Celery task engine.
Redis Strategy
Acts as high-speed messaging broker queue and storage backing for Celery workers.
Async Setup
Complete non-blocking client interaction. HTTP calls return in 10ms; calculations run on backends.
Scaling Concerns
Queue clogging: implement separate queues for fast vs slow tasks. Scale consumer pods independently.
Observability
Flower dashboard to monitor queue health. Active task tracing metrics.
API Endpoints
  • • POST /tasks/process — dispatch job, returns task_id
  • • GET /tasks/status/{id} — poll task completion + telemetry
Dev Checks
  • • Launch Celery worker container in Docker Compose
  • • Set retry policies on worker tasks for unstable API failures
  • • Integrate polling loop reporting granular task state
Prod Stack Concern Task durability: ensure workers retry crashed file processes. Clean up system file paths safely.
Project 4: Real-Time Chat Backend
MEDIUM FastAPI · Redis · Docker-Compose
Subscribing routing engine. Handles persistent WebSocket pipelines, using Redis Pub/Sub to mirror chat streams horizontally across multiple backend nodes.
Redis Strategy
Redis Pub/Sub coordinates multiple running backend nodes to synchronize chat rooms globally.
Async Setup
Asyncio loops coordinate reading and writing on sockets simultaneously.
Scaling Concerns
Memory leaks on dangling handles. Max connection bottlenecks: keep state metadata in memory low.
Observability
Connection lifetime telemetry tracking socket closures.
API Endpoints
  • • WS /ws/chat/{room_id} — persistent duplex pipeline
Dev Checks
  • • Implement keepalive ping loops to close inactive connections
  • • Write Pub/Sub listeners in asyncio task loops
  • • Set security middleware checking auth query params
Prod Stack Concern WebSocket handshaking validation, token checking over connection requests, standard connection heartbeats.
Project 5: Semantical RAG System
HARD FastAPI · PostgreSQL · pgvector · Google GenAI SDK
Vector ingestion pipeline. Ingests PDFs, chunks structurally, saves similarity indexes inside pgvector, and injects context into Gemini prompt loops.
Redis Strategy
Semantic caching of similarity queries — resolving repeating questions instantly without hitting LLM API gateways.
Async Setup
Async streams yield tokens one by one (Server-Sent Events) for real-time generative layouts.
Scaling Concerns
Embedding throughput limitations. HNSW vector index tuning. Prompt token size allocations.
Observability
Token calculation logging. Grounding metrics measuring retrieved relevance distances.
API Endpoints
  • • POST /documents/ingest — process incoming documents
  • • POST /chat/query — query pipeline, returns streamed context-grounded AI text
Dev Checks
  • • Build semantic chunker slicing paragraphs at real boundary intersections
  • • Expose search routes matching queries with pgvector Cosine Metrics
  • • Implement conversational model interface tracking histories dynamically
Prod Stack Concern Prompt Injection attacks, document format safety, caching expirations, system constraints enforcement.
Project 6: AI Workflow System
HARD FastAPI · PostgreSQL · LangGraph · Google GenAI API · Langfuse
Functional graph state machine. Models multi-turn AI reasoning loops with strict validation schemas and automated fallbacks.
Redis Strategy
Session state variables cache, locking concurrent operations.
Async Setup
Runs concurrent LLM queries dynamically via async gathers in appropriate graph nodes.
Scaling Concerns
Concurrent state mutations: serialize graph paths. Manage API Rate Limits on multiple API hubs.
Observability
Telemetry tracking latency, step completion ratios, and dollar spend statistics on API loops.
API Endpoints
  • • POST /workflow/run — execute structured AI task, returns state node logs
  • • GET /workflow/traces — audit execution footprints and trace output logs
Dev Checks
  • • Write complete Pydantic model configurations verifying model structure parameters
  • • Configure error recovery nodes dynamically shifting tasks to fallback APIs
  • • Deploy full-stack gateway dashboard to audit execution histories
Prod Stack Concern Fail-safe state recovery if an intermediate step crashes. Auto-retries with exponential backoffs.
